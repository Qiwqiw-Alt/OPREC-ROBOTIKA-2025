## Apa itu Git & GitHub?

### 1. Git (The Tool) 
**Git** adalah sistem pengontrol versi (*Version Control System*).
* **Analogi:** Bayangkan "Save Game" di video game. Sebelum melawan bos, kamu melakukan *save*. Jika kalah, kamu bisa *load* kembali ke titik itu.
* **Fungsi:** Git mencatat setiap perubahan kode yang kamu buat di komputer kamu. Kamu bisa melihat riwayat perubahan, dan mengembalikan kode ke kondisi sebelumnya jika terjadi *error*.
* **Sifat:** Offline (berjalan di komputermu sendiri).

### 2. GitHub (The Place) 
**GitHub** adalah layanan hosting (penyimpanan) untuk file Git.
* **Analogi:** Jika Git adalah "Kamera" untuk memotret kodemu, maka GitHub adalah "Instagram" atau "Google Drive"-nya tempat kamu menyimpan dan memamerkan foto-foto tersebut.
* **Fungsi:** Tempat menyimpan kodingan secara online (cloud), tempat berkolaborasi dengan orang lain, dan portofolio untuk dilirik perekrut kerja.
* **Sifat:** Online (berbasis web).

## Kenapa Harus Belajar Ini?

1.  **Mesin Waktu:** Salah hapus kode? Tenang, bisa dikembalikan ke versi sebelumnya dalam hitungan detik.
2.  **Kolaborasi:** Mengerjakan satu proyek bersama 10 orang tanpa takut file saling menimpa (*overwrite*).
3.  **Standar Industri:** Hampir semua perusahaan teknologi mewajibkan karyawannya bisa Git.
4.  **Aman:** Laptop rusak/hilang? Kodingan aman tersimpan di GitHub.
